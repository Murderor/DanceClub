# Dance Club Manager - Структура проекта

## 📁 Общая структура файлов

```
Danceclub/
├── index.html          # Главная страница
├── style.css           # Стили приложения
└── js/                 # JavaScript модули
    ├── config.js       # Конфигурация и глобальные переменные
    ├── auth.js         # Авторизация и регистрация
    ├── profile.js      # Управление профилем пользователя
    ├── availability.js # Управление доступностью (свободное время)
    ├── events.js       # Просмотр расписания репетиций
    ├── admin.js        # Админ панель (управление коллективом)
    └── main.js         # Главный модуль приложения
```

## 🗄️ Структура базы данных (Supabase)

### Таблица `profiles`
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | ID пользователя (связь с auth.users) |
| name | TEXT | Имя пользователя |
| full_name | TEXT | Полное имя |
| email | TEXT | Email пользователя |
| role | TEXT | Роль: 'member' или 'admin' |
| unique_code | VARCHAR(5) | Уникальный 5-символьный код пользователя |
| group_id | UUID | ID коллектива (внешний ключ) |
| created_at | TIMESTAMP | Дата создания |

### Таблица `groups`
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID коллектива |
| name | TEXT | Название коллектива |
| invite_code | VARCHAR(10) | Код приглашения в коллектив |
| created_by | UUID | ID создателя коллектива |
| created_at | TIMESTAMP | Дата создания |

### Таблица `group_members`
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID записи |
| group_id | UUID | ID коллектива |
| user_id | UUID | ID пользователя |
| joined_at | TIMESTAMP | Дата присоединения |

### Таблица `availability`
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID записи |
| user_id | UUID | ID пользователя |
| day_of_week | INT | День недели (0=ПН ... 6=ВС) |
| start_time | TIME | Время начала |
| end_time | TIME | Время окончания |
| created_at | TIMESTAMP | Дата создания |

### Таблица `events`
| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный ID события |
| title | TEXT | Название репетиции |
| description | TEXT | Описание репетиции |
| start_time | TIMESTAMP | Дата и время начала |
| end_time | TIMESTAMP | Дата и время окончания |
| created_by | UUID | ID создателя события |
| group_id | UUID | ID коллектива (внешний ключ) |
| created_at | TIMESTAMP | Дата создания |

## 🔧 Функции базы данных

### `generate_unique_code()`
Генерирует уникальный 5-символьный код для пользователя

### `get_user_role()`
Возвращает роль текущего пользователя

### `handle_new_user()`
Триггер, создающий профиль при регистрации нового пользователя

## 🔐 Политики безопасности (RLS)

### Таблица `profiles`
- `Profiles are viewable by authenticated users` - просмотр для авторизованных
- `Users can update own profile` - обновление своего профиля
- `Users can insert own profile` - создание своего профиля

### Таблица `groups`
- `Users can view groups` - просмотр всех групп
- `Admins can create groups` - создание групп только админами
- `Creator or admin can update groups` - обновление создателем или админом
- `Admins can delete groups` - удаление только админами

### Таблица `group_members`
- `allow_select_all` - просмотр разрешен всем
- `allow_admin_all` - все операции только для админов

### Таблица `availability`
- `Availability viewable by authenticated users` - просмотр для авторизованных
- `Users can manage own availability` - управление своей доступностью

### Таблица `events`
- `Events viewable by group members` - просмотр для участников группы
- `Only admin can create events` - создание только админами
- `Only admin can update events` - обновление только админами
- `Only admin can delete events` - удаление только админами

## 🎯 Основные функции приложения

### Для всех пользователей
1. **Регистрация и вход** - через email/пароль
2. **Управление профилем**
   - Просмотр уникального кода
   - Изменение имени
   - Вступление в коллектив по коду приглашения
3. **Управление доступностью**
   - Добавление временных слотов по дням недели
   - Удаление слотов
4. **Просмотр расписания**
   - Список предстоящих репетиций коллектива

### Для администраторов
1. **Управление коллективом**
   - Создание коллектива
   - Генерация кода приглашения
   - Просмотр списка участников
   - Добавление участников по их уникальному коду
2. **Управление репетициями**
   - Календарь с существующими событиями
   - Создание новых репетиций
   - Проверка доступности участников
   - Удаление репетиций

## 📝 Примечания

1. При регистрации пользователь получает автоматически сгенерированный уникальный 5-символьный код
2. Первый пользователь не становится автоматически админом - роль нужно назначить вручную через SQL
3. Все события (репетиции) привязаны к коллективу
4. Пользователь может состоять только в одном коллективе
5. Уникальный код пользователя используется для добавления в коллектив
6. Код приглашения коллектива используется для вступления

## 🐛 Известные проблемы

1. Иногда возникают проблемы с RLS политиками (бесконечная рекурсия)
2. Может потребоваться ручное добавление поля group_id в таблицу events
3. При первом запуске нужно вручную назначить админа через SQL

## 🔧 SQL для создания админа

```sql
UPDATE public.profiles 
SET role = 'admin' 
WHERE id = 'UUID_ПОЛЬЗОВАТЕЛЯ';
```
