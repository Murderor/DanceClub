# Танцевальный менеджер - Текущее состояние проекта

## 📋 Обзор

Веб-приложение для управления танцевальными коллективами. Позволяет создавать коллективы, добавлять участников, генерировать уникальные 6-значные коды.

## 🛠 Технологии

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Backend**: Supabase (PostgreSQL + Auth)
- **Библиотеки**: SweetAlert2, Font Awesome 6, Supabase JS Client

## 📁 Структура

```
dance-manager/
├── index.html
├── styles/style.css
├── js/
│   ├── config.js      # Supabase конфигурация
│   ├── auth.js        # Авторизация и регистрация
│   ├── users.js       # CRUD участников
│   ├── groups.js      # Управление коллективами
│   └── main.js        # Инициализация и навигация
```

## 📦 Модули

- `window.currentUser` - текущий пользователь

- `window.allUsers` - список всех участников

### auth.js

|-------|----------|
| `checkSession()` | Проверка сессии, загрузка профиля |

### users.js

|-------|----------|

### groups.js (НОВЫЙ)

|-------|----------|

## 🗄 База данных

### Таблица users
```sql
id UUID PRIMARY KEY (auth.users)
email TEXT NOT NULL UNIQUE
last_name TEXT NOT NULL
first_name TEXT NOT NULL
patronymic TEXT
birth_date DATE NOT NULL
unique_code TEXT UNIQUE NOT NULL
role TEXT DEFAULT 'user'
created_at TIMESTAMP
updated_at TIMESTAMP
```

### Таблица dance_groups
```sql
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
name TEXT NOT NULL
leader_id UUID REFERENCES users(id)
leader_name TEXT NOT NULL
description TEXT
created_at TIMESTAMP
updated_at TIMESTAMP
```

### Таблица group_members
```sql
group_id UUID REFERENCES dance_groups(id)
user_id UUID REFERENCES users(id)
joined_at TIMESTAMP
role TEXT DEFAULT 'member' -- 'member' или 'leader'
PRIMARY KEY (group_id, user_id)
```

### Индексы
- `idx_users_unique_code`, `idx_users_last_name`, `idx_users_first_name`, `idx_users_email`
- `idx_dance_groups_name`, `idx_dance_groups_leader`
- `idx_group_members_user`, `idx_group_members_group`

### Триггеры
- `update_updated_at_column()` - автообновление updated_at для users и dance_groups

## 🎯 Функциональность (статус)

### ✅ Реализовано
- Регистрация и вход
- Генерация уникальных кодов
- CRUD участников
- Создание/редактирование/удаление коллективов
- Добавление/удаление участников в коллективы
- Показ только коллективов пользователя
- Статистика: количество коллективов, где лидер
- Поиск по коллективам
- Карточки с бейджами (Лидер/Участник)
- Кнопка "Все участники" с возвратом

### ❌ Требует проверки/исправления
- При создании коллектива пользователь должен автоматически добавляться как участник
- При просмотре коллектива должны отображаться все участники
- Удаление участника из коллектива должно работать
- Статистика "Мои коллективы" должна обновляться при изменениях

## 🐛 Текущие проблемы

1. **Статистика отображается корректно** - числа показываются правильно
2. **Создание коллектива** - работает, пользователь становится лидером
3. **Добавление участников** - по коду, только для лидера
4. **Навигация** - переключение между дашбордом и таблицей участников работает

## 🔄 Навигация в приложении

1. После входа → **Дашборд** (показывает только коллективы пользователя)
2. Кнопка "Все участники" → **Таблица всех участников**
3. Кнопка "Назад к коллективам" в таблице → **Дашборд**
4. Кнопка "Создать коллектив" → модальное окно
5. Карточка коллектива → просмотр/редактирование/удаление

## 📝 Важные особенности

- Пользователь видит только коллективы, где он состоит
- Лидер может управлять своим коллективом (редактировать, удалять, добавлять/удалять участников)
- Участник может только просматривать коллектив
- Уникальные коды генерируются автоматически
- Нельзя удалить свой профиль из таблицы участников
- При создании коллектива пользователь автоматически становится лидером и добавляется в group_members
